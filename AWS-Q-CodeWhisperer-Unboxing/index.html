開箱工程師兩大利器：Amazon Q、AWS CodeWhisperer

前言

今天要來介紹 Amazon Q + AWS CodeWhisperer，並進行簡單的小開箱試用 ✨
主要是為了 AWS Educate Taiwan 雲端校園大使計畫的加分題！
期待能夠在 AWS 中學習新技術並推廣所學！

簡介

Amazon Q

據官網所說，這是一種新型的生成式 AI 助理，專門用於滿足辦公需求，且可以根據客戶業務進行訂製。基於客戶自身的資訊儲存庫、程式碼和企業系統，客戶可以快速獲得複雜問題的相關答案、生成內容並採取行動。

其中最特別的是，可以根據客戶業務進行訂製。基於客戶自身的資訊儲存庫、程式碼和企業系統，客戶可以快速獲得複雜問題的相關答案、生成內容並採取行動。

因此 Amazon Q 是為了滿足企業級客戶嚴格要求而特別打造，可以根據企業既有的人員、角色和權限對每個用戶交互進行客製化。

AWS CodeWhisperer

根據官網所述，這是一款適用於 IDE 和命令列的 AI 驅動的生產力工具，他可以根據現有的程式碼和註釋生成建議代碼。

AWS CodeWhisperer 有以下幾個特點：

  1. reference log
    CodeWhisperer 可以標記或篩選與公開可用程式碼類似的程式碼建議。取得關聯開放原始碼專案的儲存庫 URL 和授權，以便您更輕鬆地檢閱並新增歸因。

  2. 尋找安全性漏洞
    掃描程式碼以識別難以發現的安全漏洞，並取得程式碼建議以協助修復已識別的問題。採用生成式 AI 技術的程式碼建議可根據您的應用程式程式碼量身打造，因此可以快速、自信地接受修正，而專注於更高價值的工作。

  3. 量身打造的程式碼生成器
    可以透過了解內部程式庫、API、套件、類別和方法，進而給出更相關的建議以達到加速開發。

  4. 搭配 Amazon Q 提高生產力

    能夠透過反白的方式，使用 Amazon Q 解釋程式碼或是找出 bug，看起來最吸引人的便是將應用程式升級和遷移至最新版本，這對於專案使用一堆整天在更新的庫，後期要針對更新進行修正來說，是一大福音呢！

而目前支援 IDE 有：VS Code、Intellij IDEA、Pycharm、AWS Cloud9、AWS Lambda Console
支援語言有：Python、Java、JavaScript、TypeScript、C#

安裝

安裝真的可以說是非常簡單，幾乎不需要懶人包。

首先你需要準備一個 IDE，我以 Visual Studio Code 為演示

  1. 進入 vscode，打開 extension 延伸模組，輸入 AWS Toolkit：

  2. 安裝 AWS Toolkit

  3. 點擊最下方的 AWS 或是左側的 AWS，點「connect with AWS Builder ID」

  4. 若沒有 AWS Builder 帳號，會自動導向註冊，開始註冊！

  5. 註冊完就可以順利使用啦 ✨

結語

我認為在使用上，Amazon Q + CodeWhisperer 是可以幫助我的，不管是加速開發抑或是程式碼解釋，不會出現牛頭不對馬嘴的解釋字句，或是拖慢開發速度的 bug code，但還是仰賴工程師本身的基礎能力，讓工程師專注於優化、測試上，而不用花費大量時間在同樣性質的工作上。

另外，很方便的地方是只需要不到五分鐘的註冊，就可以在 IDE 上免費使用，完全無痛使用，對於 amateur or professional Dev 都是一大利器（最重要是面向於企業方面）。更重要的是，還能根據專案架構，提供輔助和修正，這也是企業內部工程師最需要的幫助。

經過這陣子的試用，我個人是看好的，尤其是會掃描專案後再給你 suggestion， 相當於一位看過完整程式碼的工程師從旁輔助你，可以省下大量時間和精力。

但這兩款畢竟推出時間不長，仍然有許多地方值得改進，例如：支援程式語言過少、Amazon Q 不支援繁體中文。
